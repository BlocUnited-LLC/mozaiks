Overview & Quick Start

Understand the two-file config system and get a branded dev environment running in under five minutes.

How it works

The chat-ui template reads all brand and layout configuration from two static JSON files served from the app's public directory. At startup, themeProvider.js fetches both files in parallel, merges them into a single theme object, and applies CSS custom properties across the entire app.

brand.json — Visual Identity

Colors, typography, shadows, and asset filenames. Controls what the UI looks like.
→ Full reference

ui.json — Chrome Config

Header actions, profile menu, notifications, footer links, chat modes. Controls what appears in the shell.
→ Full reference

If a config file or individual key is missing, the runtime falls back to the built-in BARE_FALLBACK_THEME and logs a console.warn. The app always renders — missing config is visible in DevTools, not in crashes.

ℹ️

Zero code changes required for most customizations. Edit JSON, drop in assets, done.

File layout

Template directory
packages/frontend/chat-ui/
  template/                        ← copy this into your app
    vite.config.js                 # publicDir → "brands/public"
    package.json
    src/
      main.jsx
      App.jsx
    brands/
      public/                      ← Vite publicDir — served at /
        brand.json                 # Visual identity
        ui.json                    # UI chrome config
        assets/
          mozaik_logo.svg          # brand.json → assets.logo
          mozaik.png               # brand.json → assets.wordmark
          chat_bg_template.png     # brand.json → assets.backgroundImage
          profile.svg              # ui.json → profile.icon
          notifications.svg        # ui.json → notifications.icon
          sparkle.svg              # ui.json → header.actions icon
          settings.svg             # ui.json → profile.menu icons
          logout.svg               # ui.json → profile.menu icons

Because brands/public/ is the Vite publicDir, all files inside are served at /. A file at brands/public/assets/sparkle.svg is accessed as /assets/sparkle.svg. JSON config references use this path.

Quick start

  1. Copy the template into your app root. The template directory is your starting point — don't edit files inside packages/frontend/chat-ui/src/ directly.
  2. Install dependencies in the template directory:
    Terminal
    cd template
npm install
  3. Edit brands/public/brand.json — swap in your colors, fonts, and asset filenames. See Step 2 for the full spec.
  4. Edit brands/public/ui.json — configure your header actions, profile dropdown menu, and footer links. See Step 3.
  5. Drop your assets into brands/public/assets/. All icon values in JSON must be filenames (e.g. "bell.svg"). See Step 4.
  6. Wire up navigation and auth via the onAction prop. See Step 5.

Start the dev server

Terminal
cd packages/frontend/chat-ui/template
npm run dev
# → http://localhost:3000

Changes to brand.json and ui.json are picked up on browser reload. Asset file changes (SVGs, images) are reflected immediately because they are served directly from the public directory.

⚠️

Icon values must be filenames. Values like "bell" or "sparkle" are not built-in tokens. Always use filenames: "bell.svg", "sparkle.svg". A non-filename value logs a console warning and renders nothing.
