brand.json

Define your visual identity — colors, typography, shadows, and asset filenames. No UI structure or menu configuration belongs here.

📁

File location: brands/public/brand.json — served at /brand.json

Full example

brands/public/brand.json
{
  "name": "YourApp",
  "tagline": "AI-Powered Everything",

  "assets": {
    "logo":            "logo.svg",
    "wordmark":        "wordmark.png",
    "favicon":         "favicon.png",

name & tagline

FieldTypeDescription
name string required App display name. Used in page titles and screen reader labels throughout the shell.
tagline string optional Short tagline shown in login screens or splash contexts. Not shown in the main shell by default.

assets

Filenames for images served from /assets/. Do not include the path prefix — just the filename.

FieldTypeDescription
assets.logo string required Primary logomark (SVG or PNG). Used in the loading spinner and any context requiring the logo icon. Example: "logo.svg"
assets.wordmark string optional Full wordmark / logotype image — used where horizontal layout allows more space.
assets.favicon string optional Favicon filename. Injected into <link rel="icon"> at app boot.
assets.backgroundImage string optional Chat area background image. Falls back to chat_bg_template.png if missing. Example: "chat_bg.png"

fonts

Three font slots: body, heading, logo. Each is independent.

FieldTypeDescription
fonts.*.family string CSS font-family value for this slot (e.g. "Rajdhani").
fonts.*.fallbacks string Comma-separated CSS fallback fonts appended after the primary family.
fonts.*.googleFont string optional Full Google Fonts URL. Injected as a <link rel="stylesheet"> at boot. Omit if using a local font.
fonts.*.localFont boolean optional Set true for self-hosted fonts. Requires src to be set.
fonts.*.src string optional Path to the font file from the public root (e.g. "/fonts/MyFont.otf"). Used to generate a @font-face block.
fonts.*.tailwindClass string optional Tailwind utility class applied to the relevant element (e.g. "font-heading").
💡

Place self-hosted font files in brands/public/fonts/ — they'll be served at /fonts/<file>.

colors

All color entries are applied as CSS custom properties (e.g. --color-primary, --color-primary-light). Components reference these vars, so changing a token here affects the entire UI.

Semantic tokens

Each of primary, secondary, accent, success, warning, error uses this shape:

Color token shape
{
  "main":  "#06b6d4",   // Primary use
  "light": "#67e8f9",   // Hover, highlight, tint
  "dark":  "#0e7490",   // Active, pressed state
  "name":  "cyan"       // Human-readable label (informational)
}

Background, border, text

KeySub-keys
backgroundbase · surface · elevated · overlay
bordersubtle · strong · accent
textprimary · secondary · muted · onAccent

shadows

Named box-shadow values applied as CSS custom properties (e.g. --shadow-primary). Accepts any valid CSS box-shadow string.

KeyTypical use
primaryGlow on primary-colored elements (cyan by default)
secondaryGlow on secondary-colored elements (violet)
accentGlow on accent elements (amber)
elevatedGeneral elevated card shadow
focusFocus ring for keyboard navigation
