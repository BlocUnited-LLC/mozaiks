Wiring & Backend

Connect the chat-ui shell to your app: handle menu actions, supply authenticated user context, and integrate your backend API endpoints.

The onAction contract

Every interactive element in the header and profile dropdown dispatches through a single onAction(action, item) callback that you supply to the shell. This is the integration point between declarative config and your app's runtime.

Signature
/**
 * @param {string} action  - The action string from ui.json (e.g. "navigate", "signout", "discover")
 * @param {object} item    - The full menu/action item object from ui.json
 */
function onAction(action, item) { … }
Full handler example
import { useNavigate } from 'react-router-dom';
import { useAuth }     from './auth/useAuth';

export function useActionHandler() {
  const navigate = useNavigate();
  const { signOut } = useAuth();

  return async function onAction(action, item) {
    switch (action) {

      case 'navigate':
        navigate(item.href);
        break;

      case 'signout':
        await signOut();
        navigate('/login');
        break;

      // Custom actions declared in ui.json header.actions
      case 'discover':
        navigate('/discover');
        break;

      case 'new-workspace':
        navigate('/workspaces/new');
        break;

      default:
        console.warn('[onAction] Unhandled action:', action, item);
    }
  };
}
💡

The action string is always the "action" field value from ui.json. For header buttons the "id" is passed as the action. Keep them consistent and human-readable.

User context object

Pass an authenticated user object to the shell and it populates the profile button display name, avatar, and sublabel. The shell does not own authentication state — you supply it.

User context shape
const userContext = {
  displayName: "Maya Reyes",       // String — shown in profile dropdown header
  email:       "maya@yourapp.com", // String — shown as secondary line if no sublabel override
  photoURL:    "https://…/photo",  // String | null — avatar image URL

  // Optional extended fields — not displayed by default but available via context
  uid:         "user_abc123",
  plan:        "pro",
  role:        "admin"
};
FieldTypeShell usage
displayName string Overrides profile.defaultLabel from ui.json when the user is authenticated.
email string optional Shown as dropdown subtitle when profile.sublabel is not set in ui.json.
photoURL string | null Rendered as the profile avatar image. Falls back to the avatar icon on null.

Avatar render priority

  1. 1
    User photo — userContext.photoURL when present and loads successfully
  2. 2
    Generated initials — derived from userContext.displayName if photo fails to load
  3. 3
    Fallback icon — the profile.icon SVG from ui.json

Auth provider integration

The shell accepts user context via a prop or context provider. The recommended pattern wraps the shell with your auth context and maps the authenticated user to the expected shape:

Firebase Auth example
import { useAuthState } from 'react-firebase-hooks/auth';
import { auth }         from './firebase';
import { ChatShell }    from '@mozaiks/chat-ui';
import { useActionHandler } from './hooks/useActionHandler';

export function App() {
  const [firebaseUser] = useAuthState(auth);
  const onAction = useActionHandler();

  const userContext = firebaseUser
    ? {
        displayName: firebaseUser.displayName,
        email:       firebaseUser.email,
        photoURL:    firebaseUser.photoURL,
        uid:         firebaseUser.uid
      }
    : null;

  return (
    <ChatShell
      userContext={userContext}
      onAction={onAction}
    />
  );
}
ℹ️

Any auth provider works — Supabase, Clerk, Auth0, custom JWT. Map their user object to the { displayName, email, photoURL } shape before passing it down.

Backend API integration

Chat messages, workflow triggers, and artifact creation route through the Mozaiks core API. Configure the API base URL and any auth headers via the shell's config:

Runtime config
import { ChatShell, configureMozaiks } from '@mozaiks/chat-ui';

// Call once at app bootstrap, before rendering the shell
configureMozaiks({
  apiBaseUrl:  'https://api.yourapp.com',
  getAuthToken: async () => {
    // Return the current bearer token for outbound API requests
    return await auth.currentUser.getIdToken();
  }
});

export function App() {
  return <ChatShell … />;
}
OptionTypeDescription
apiBaseUrl string required Root URL for all Mozaiks API calls (no trailing slash).
getAuthToken function → Promise<string> required Called before each request. Return the bearer token; it's added as Authorization: Bearer <token>.

Custom action names

Any string value in "action" fields (profile menu items, header actions) is forwarded to your onAction handler as-is. You are not limited to built-in values.

ui.json — custom action
{
  "id":     "open-feedback",
  "label":  "Send Feedback",
  "icon":   "feedback.svg",
  "action": "open-feedback"
}
onAction handler
case 'open-feedback':
  setFeedbackDrawerOpen(true);
  break;
🎉

You're done! At this point your brand is applied, your assets are in place, and the shell is wired to your auth and backend. Go build something.
