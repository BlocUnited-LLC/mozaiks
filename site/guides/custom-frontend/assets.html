ActionIcon rules


    

      All icon references in ui.json — header.actions[*].icon,
      profile.menu[*].icon, notifications.icon, and
      profile.icon — are passed to the ActionIcon component.
    


    

      ⚠️
      

        
Filename required. The value must contain a dot (e.g. "sparkle.svg").

        
If you pass a bare name like "sparkle", ActionIcon logs a console warning and renders
           null — no icon appears. This is intentional; token strings are not supported.

      

    


    
How ActionIcon resolves an icon value:


    
    
      
  1. If the value contains a dot, the icon is treated as a filename.
    2. 
      
  2. The component requests /assets/<filename> from the brand server.
    3. 
      
  3. The SVG source is fetched, sanitized, and rendered inline — inheriting the current color via fill="currentColor".
    4. 
      
  4. Size is controlled by the parent context (header toolbar, menu row, etc.).
    5. 
    


    
    
SVG format requirements


    
Icons rendered via ActionIcon must follow these rules for correct color inheritance:
RuleWhy
Use fill="currentColor" So the icon inherits the button/text color from CSS. Hard-coded hex values break theme switching.
Set viewBox="0 0 24 24" Standard 24 × 24 viewport ensures predictable scaling. Other sizes are supported but 24 × 24 is expected.
No width / height attributes on the <svg> tag The parent container controls rendered size. Fixed dimensions override CSS sizing.
Single-color paths only Multi-color icons won't respond to currentColor. Use a raster image instead for multi-color logos.

Minimal valid SVG example

assets/sparkle.svg
<svg viewBox="0 0 24 24" fill="currentColor" xmlns="http://www.w3.org/2000/svg">
  <path d="M12 2L9.5 9.5 2 12l7.5 2.5L12 22l2.5-7.5L22 12l-7.5-2.5Z"/>
</svg>

PNG & WebP

Raster images (PNG, WebP, JPEG) are used for the chat background, wordmark, and favicon — not for icon buttons. They are referenced directly via <img> tags and do not go through ActionIcon.

AssetFormatNotes
assets.backgroundImage PNG / WebP Chat area background. Recommended resolution: 1440 × 900 or larger. Falls back to chat_bg_template.png if the file is absent.
assets.wordmark PNG / SVG Logotype. Transparent background recommended. Will be rendered at natural aspect ratio up to a max height.
assets.favicon PNG / ICO Injected as <link rel="icon">. 32 × 32 or 64 × 64 PNG works across all browsers.
